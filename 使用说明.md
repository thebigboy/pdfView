# PDF电子发票解析程序

这是一个使用Java开发的PDF电子发票解析工具，可以自动提取PDF格式电子发票中的关键信息。

## 功能特性

- ✅ 解析PDF格式的电子发票文件
- ✅ 自动提取发票号码、发票代码、开票日期
- ✅ 提取销售方和购买方信息（名称、纳税人识别号）
- ✅ 解析金额信息（价税合计、税额、不含税金额）
- ✅ 支持发票明细项目解析
- ✅ 输出结构化的发票信息
- ✅ 可选择保存解析结果为JSON格式
- ✅ 支持批量处理多个文件
- ✅ 交互式命令行界面

## 技术栈

- **Java 8+**: 核心开发语言
- **Apache PDFBox**: PDF文件解析库
- **Jackson**: JSON序列化/反序列化
- **SLF4J**: 日志框架
- **Maven**: 依赖管理和构建工具

## 快速开始

### 1. 编译项目

```bash
mvn clean compile
```

### 2. 运行程序

#### 交互式模式
```bash
mvn exec:java -Dexec.mainClass="com.invoice.InvoiceParserMain"
```

#### 命令行模式（指定文件）
```bash
mvn exec:java -Dexec.mainClass="com.invoice.InvoiceParserMain" -Dexec.args="path/to/invoice.pdf"
```

#### 批量处理
```bash
mvn exec:java -Dexec.mainClass="com.invoice.InvoiceParserMain" -Dexec.args="invoice1.pdf invoice2.pdf invoice3.pdf"
```

### 3. 构建JAR包

```bash
mvn clean package
java -jar target/pdf-invoice-parser-1.0.0.jar invoice.pdf
```

### 程序化使用
```java
import com.invoice.PdfInvoiceParser;
import com.invoice.InvoiceInfo;
import java.io.File;

public class Example {
    public static void main(String[] args) throws Exception {
        PdfInvoiceParser parser = new PdfInvoiceParser();
        File pdfFile = new File("invoice.pdf");
        
        InvoiceInfo invoice = parser.parsePdfInvoice(pdfFile);
        
        System.out.println("发票号码: " + invoice.getInvoiceNumber());
        System.out.println("总金额: " + invoice.getTotalAmount());
        System.out.println("开票日期: " + invoice.getInvoiceDate());
    }
}
```

## 支持的发票格式

程序主要针对中国大陆地区的电子发票PDF格式进行优化，支持：

- 增值税普通发票
- 增值税专用发票
- 电子普通发票
- 电子专用发票

## 解析字段说明

| 字段名称 | 字段说明 | 数据类型 |
|---------|----------|----------|
| invoiceNumber | 发票号码 | String |
| invoiceCode | 发票代码 | String |
| invoiceDate | 开票日期 | LocalDate |
| sellerName | 销售方名称 | String |
| sellerTaxNumber | 销售方纳税人识别号 | String |
| buyerName | 购买方名称 | String |
| buyerTaxNumber | 购买方纳税人识别号 | String |
| totalAmount | 价税合计金额 | BigDecimal |
| taxAmount | 税额 | BigDecimal |
| amountWithoutTax | 不含税金额 | BigDecimal |
| remarks | 备注 | String |
| items | 发票明细项目列表 | List<InvoiceItem> |

## 注意事项

1. **PDF格式要求**: 程序主要适用于文本型PDF，对于扫描版PDF效果可能不佳
2. **字符编码**: 确保PDF文件使用正确的中文编码
3. **发票格式**: 不同地区或不同版本的发票格式可能需要调整正则表达式
4. **性能考虑**: 大批量处理时建议分批进行
5. **错误处理**: 程序会记录详细日志，解析失败时请检查日志信息

## 扩展开发

如果需要支持新的发票格式或字段，可以：

1. 修改 `PdfInvoiceParser.java` 中的正则表达式模式
2. 在 `InvoiceInfo.java` 中添加新的字段
3. 更新相应的解析方法

## 常见问题

### Q: 解析结果不准确怎么办？
A: 检查PDF文件是否为文本型，调整相应的正则表达式模式。

### Q: 如何支持新的发票格式？
A: 分析新格式的文本布局，修改或添加相应的正则表达式。

### Q: 程序运行出错怎么办？
A: 检查依赖是否正确安装，查看控制台日志信息。
