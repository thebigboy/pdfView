# PDF电子发票解析器 - 改进说明

## 问题分析

根据您提供的解析结果，原始解析器在处理您的PDF发票时遇到了以下问题：

1. **销售方和购买方信息解析失败** - 正则表达式模式不匹配实际的文本格式
2. **金额信息解析不完整** - 价税合计、税额、不含税金额未正确提取
3. **发票明细解析格式不匹配** - 明细项目的格式与预期不符

## 改进方案

### 1. 优化正则表达式模式

**原始模式问题:**
```java
// 原始的销售方名称匹配模式
private static final Pattern SELLER_NAME_PATTERN = Pattern.compile("销售方[：:]?\\s*名称[：:]\\s*([^\\n\\r]+)");
```

**改进后的模式:**
```java
// 改进后的解析方法 - 直接匹配文本中的模式
Pattern namePattern = Pattern.compile("名称[：：]([^\\n\\r]+?)\\n");
Pattern taxPattern = Pattern.compile("统一社会信用代码/纳税人识别号[：：]([A-Z0-9]{15,20})");
```

### 2. 改进金额解析逻辑

**针对您的发票格式优化:**
- 价税合计：从 "价税合计（大写）（小写）764.00¥" 中提取
- 不含税金额和税额：从 "合计 720.75¥ 43.25¥" 中提取

### 3. 改进明细项目解析

**针对您的明细格式:**
```
*经纪代理服务*代订住 次 1 720.75 720.75 6% 43.25
```

使用新的正则表达式：
```java
Pattern itemPattern = Pattern.compile("\\*([^*]+)\\*([^\\n\\r]*?)\\s+(\\S+)\\s+(\\d+(?:\\.\\d+)?)\\s+(\\d+(?:\\.\\d+)?)\\s+(\\d+(?:\\.\\d+)?)\\s+(\\d+(?:\\.\\d+)?%)\\s+(\\d+(?:\\.\\d+)?)");
```
